# Режим: Оценка юридических рисков

Выяви скрытые обязательства и оцени каждый риск по матрице серьёзность × вероятность (1-25). См. `reference/frameworks.md`.

## Процесс
1. **Выяви риски** - пройди документ/сделку и собери скрытые обязательства, неочевидные последствия, пробелы.
2. **Оцени каждый:** серьёзность S(1-5) × вероятность P(1-5) = балл (1-25) → зона 🟢/🟡/🟠/🔴.
3. **Открой нужный `reference/`** для правовой части (gdpr / eu-ai-act / germany / ukraine) и **сверь по JIT**.
4. **Двойная маркировка** - норма отдельно, применение к кейсу отдельно.

## Вывод
Таблица/список, отсортировано 🔴→🟢. Каждый риск одной строкой:
- **Суть риска** + **балл с раскладкой** (напр. S4 × P3 = 12, 🟠) + **рекомендуемое действие** + **что согласовать/исправить до подписания**.
- Сверху - сводка: сколько 🔴/🟠, главный вывод.
- Эскалация при критических (16-25) и при триггерах из `SKILL.md`.

## Типовые сценарии
- **Договор с вендором** - неограниченная ответственность, индемнити, права на данные, выход.
- **M&A / поглощение** - незакрытые судебные дела, скрытые обязательства, проблемы после слияния, due diligence по топ-вендорам цели.
- **Выход на новый рынок** (напр. Украина → ЕС, или ЕС → новая страна) - регуляторные, privacy-, договорные риски; локальные требования.
- **Риски ИС** - open-source лицензии (напр. AGPL в проприетарном продукте - 🔴), чистота прав, передача прав в договорах.
- **Трудовые** - переквалификация подрядчиков в штат (misclassification - актуально для Германии и Украины), неконкуренция, увольнения.
- **Privacy/данные** - см. `gdpr.md` red flags; трансфер, основания, сроки реакции на инциденты.
